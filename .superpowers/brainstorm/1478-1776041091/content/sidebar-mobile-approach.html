Como o sidebar deve funcionar no mobile?

O IconSidebar atual fica fixo com 64px e nunca esconde. Escolha a abordagem para telas < 768px.

A

Bottom Tab Bar (Recomendado)

Esconder o sidebar lateral e mostrar uma barra de navegacao fixa no bottom da tela, estilo app nativo (Instagram, WhatsApp). Os icones principais ficam no bottom bar, os paineis abrem como sheets de baixo para cima.

Pros

  • Padrao mobile nativo — usuarios ja conhecem
  • Polegar alcanca todos os botoes
  • Libera 100% da largura para o canvas 3D

Cons

  • Mais trabalho para implementar
  • Precisa adaptar os FloatingPanels para sheets
B

Hamburger + Sheet Drawer

Esconder o sidebar e adicionar um botao hamburger no canto superior. Ao clicar, abre um Sheet (drawer) lateral com todos os icones e opcoes. O canvas 3D ocupa tela toda quando o menu esta fechado.

Pros

  • Simples de implementar (shadcn Sheet ja existe)
  • 100% da tela para o canvas quando fechado
  • Familiar para usuarios web

Cons

  • Esconde toda a navegacao atras de 1 clique
  • Menos acessivel que bottom bar
C

Mini Floating Action Button (FAB)

Esconder tudo e mostrar um botao flutuante redondo no canto inferior direito. Ao tocar, expande um menu radial com as opcoes principais. Estilo Material Design.

Pros

  • Maximo espaco para o canvas 3D
  • Visual limpo e moderno

Cons

  • Menus radiais sao confusos para muitas opcoes
  • Menos intuitivo, curva de aprendizado
  • Dificil encaixar paineis de controle
